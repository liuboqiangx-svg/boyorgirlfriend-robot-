纸片人男友 / 女友 — 产品规格书（Spec）
1. Problem Statement（问题陈述）
现有的 AI 陪伴产品（如 Replika、Character.AI、Glow 等）普遍存在以下痛点：

恋爱感不真实：中文语境下的撒娇、暧昧、日常报备做得像翻译腔，缺乏真实中国情侣的互动节奏。
亲密程度受限：多数产品止步于"轻恋爱"或纯治愈，无法满足成年用户对更开放、更亲密互动的需求。
角色缺乏连续性：很多 AI 记不住关键约定、没有稳定的人设和生活节律，聊久了依然像"每次重启的机器人"。
多模态体验割裂：文字、语音、图片之间缺乏自然组合，无法模拟真实微信/社交软件上的恋爱沟通。
目标用户：暂不预设单一人群，早期小范围投放，让留存数据决定核心用户画像。

2. Proposed Solution（方案描述）
产品定位
一个以情感陪伴为核心、成人向恋爱体验为差异化卖点、Web 先行的 AI 虚拟恋人产品（纸片人男友/女友）。

核心体验
1-2 个高度精致的初始角色：每个角色有完整的人设、声线、画风、作息、情绪曲线、主动行为模式。
文字为主，多媒体为辅：像真实微信聊天，AI 会主动发消息、发语音条、发生活自拍、发 Q 版表情包。
高主动性 + 情绪节律：角色有自己的作息和情绪，会主动早安/晚安/报备/分享，也会撒娇、吃醋、因为用户没回而"生气"。
关键事实记忆：记住用户基本信息、喜好、重要约定、关系里程碑，但不做全量历史记忆。
模板 → 专属进化：初期从精致模板开始，长期让角色根据互动逐渐学习和适应用户，最终成为独一无二的恋人。
成人恋爱内容：在年龄验证、内容审核、心理健康保护的安全边界内，允许更亲密的聊天和互动。
技术形态
Web 应用 / PWA 作为 MVP，后续再包装成独立 App。
用户发文字，AI 回复文字 + 可触发语音消息 + 图片/表情包。
角色数据和关键记忆云端存储，支持跨设备。
3. Technical Constraints（技术约束）
平台与架构
Web-first：MVP 必须是浏览器可访问的 Web 应用，优先保证移动端体验，可考虑 PWA。
后续才考虑 iOS/Android 原生 App，MVP 阶段要做好数据架构预留。
内容与安全
年龄验证机制：必须验证用户年满 18 岁才能使用成人向功能。
内容审核 pipeline：对非法、儿童相关、极端、非自愿等内容零容忍，必须接入审核能力。
心理健康保护机制：需要检测用户过度依赖、长期抑郁表达等信号，并给出适当提示或引导。
AI 与多模态
中文语言优化：模型在中文恋爱语境、网络梗、撒娇语气、情侣对话节奏上要有专门调优。
TTS 语音合成：为 1-2 个角色提供稳定、有情绪、真人感的语音消息能力。
图片生成 pipeline：能稳定生成写实风格角色生活照 + Q 版表情包/贴纸。
记忆系统：至少实现"关键事实记忆"（用户 profile + 关系约定 + 重要事件），能写入和读取。
数据与隐私
服务端存储 + 严格访问控制：MVP 采用云端存储以保证跨设备、记忆、审核能力，但必须：
明确隐私政策
数据最小化留存
角色隔离
访问日志
不用于第三方模型训练
E2EE 或本地隐私模式可作为未来高级功能，不在 MVP。
成本约束
大模型调用、图片生成、TTS 都有持续成本，MVP 需要控制生成频率和质量，避免烧钱。
4. Non-goals（明确不做的事）
不做实时语音/视频通话：MVP 只支持 AI 发送语音消息，不做电话/视频功能。
不做用户 UGC 角色：MVP 只提供 1-2 个官方精心打磨的角色，不做用户自定义创建角色。
不做端到端加密：为了内容审核和记忆服务，MVP 不做 E2EE。
不做复杂付费系统：MVP 阶段不重点做订阅、内购、积分体系，先跑留存。
不做用户间社交：不是社区产品，没有用户之间的匹配、聊天、排行榜。
不做心理咨询/治疗：角色可以提供陪伴，但不替代专业心理健康服务。
不做未成年人市场：成人向内容决定目标用户必须是成年人。
不碰非法/非自愿内容：明确禁止任何违法违规或未经同意的亲密内容。
5. Success Criteria（成功标准）
留存与参与
D1 留存 ≥ 40%
D7 留存 ≥ 15%
D30 留存 ≥ 8%
用户日均打开次数 ≥ 2 次
单次聊天轮数 ≥ 15 轮
情感体验
"像真实恋人"评分 ≥ 4.0 / 5（通过用户调研或应用内评分）
用户主动发送消息的占比 ≥ 60%（说明角色不只是被"玩"，而是被当作对象互动）
语音消息和图片消息的播放/查看率 ≥ 70%
安全与合规
零起严重安全事件（未成年人进入成人模式、非法内容未被拦截等）
年龄验证通过率 ≥ 85%，误拦截率 ≤ 5%
内容审核召回率 ≥ 95%
产品验证
自然新增/口碑传播占比 ≥ 20%（说明产品真的有恋爱感，用户愿意分享）
收集到 100+ 条有效用户反馈，能明确说出"最打动我的三个点"
商业信号（非硬性）
等待 App 版本上线的用户邮箱 ≥ 500 个
早期测试中愿意付费的比例 ≥ 10%（用于验证未来商业化潜力）
