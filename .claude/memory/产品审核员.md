## 🔍 审核流程

### Step 1: 用户端走查
1. 打开 https://boyorgirlfriend-robot.vercel.app/
2. 截图 → 检查 UI、加载态、空数据
3. 发送消息 → 检查 AI 回复、加载态、错误处理
4. 快捷回复 → 展开/收起是否正常
5. 未上线功能 → 是否提示"开发中"
6. 边界测试 → 空消息、超长文本、快速点击

### Step 2: 安全审计
1. **身份伪造** - deviceId 可否篡改伪装他人
2. **API 数据泄露** - 接口是否暴露 UUID/数据库字段
3. **XSS 注入** - `<script>` 等特殊字符是否转义
4. **后端校验** - 消息长度限制
5. **敏感信息** - localStorage/控制台/请求中是否有敏感数据

### Step 3: 管理员后台检查
1. 访问 `/admin` 或相关路径
2. 是否需要登录验证
3. 是否有权限分离（管理员/普通用户）
4. 敏感操作是否有日志

### Step 4: 功能完整性检查
按以下清单检查"是否有"：

| 模块 | 必需功能 | 检查点 |
|------|----------|--------|
| **用户端** | 登录/注册 | 是否需要？deviceId够用吗？ |
| **反馈系统** | 反馈入口 | 用户能否提交反馈？ |
| | 反馈处理 | 管理员能否查看处理？ |
| | 结果通知 | 处理结果是否通知用户？ |
| | 历史记录 | 反馈历史是否可查？ |
| **管理员后台** | 权限分离 | 管理员/普通用户是否分开？ |
| | 操作日志 | 敏感操作是否记录？ |
| **运维安全** | 数据备份 | 数据库是否有备份机制？ |
| | 监控告警 | 异常访问/请求是否有监控？ |
| | 敏感配置 | API Key等是否在环境变量？ |
